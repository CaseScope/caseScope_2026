Threat Descriptions:
-------------------
Disabling or Modifying Security Tools on Endpoint: The threat actor has disabled or modified security tooling in an attempt to bypass controls and further compromise the network.

Command and Control: Command and Control (C2) consists of techniques that adversaries may use to communicate with systems under their control within a victim network. Adversaries commonly attempt to mimic normal, expected traffic to avoid detection. There are many ways an adversary can establish command and control with various levels of stealth depending on the victim’s network structure and defenses.

Powershell Scriptblock - Potential Cobalt Strike Loader: A Cobalt Strike Loader is used to stage/inject the beacon for additional post-exploitation/C2 activity.

Lateral Movement: Lateral movement involves the threat actor pivoting across machines. 

Enumeration (Active directory, Network, Cloud, etc): Enumeration allows a threat actor to identity users, groups, privileges, and assets that can facilitate further ingress and compromise. 

Defense Evasion: Defense Evasion consists of techniques that adversaries use to avoid detection throughout their compromise. Techniques used for defense evasion include uninstalling/disabling security software, clearing logs, or obfuscating/encrypting data and scripts. Adversaries also leverage and abuse trusted processes to hide and masquerade their malware. 

Remediations:
-------------
Manual Remediations provided by the Huntress SOC are highly recommended remediation actions to be conducted by your team before resolving the incident in the Huntress Platform:
- Disable and/or rotate credentials for any affected user accounts.
- Deploy the Huntress agent to all endpoints in the organization to increase visibility across the network for the Huntress SOC.
- Audit local endpoint logging on affected hosts for anomalous access times or unanticipated activity.
- Ensure endpoint anti-virus solutions are deployed and are functioning properly. Audit these services for alerts or warnings on affected hosts.
- Consider enacting incident response procedures to assist in fully scoping this security incident.
- Audit network perimeter device logs such as firewalls, remote desktop services, etc. for anomalous/unauthorized access.
- Reboot the host to terminate malicious processes & sever any potentially outstanding connections. 
- Consider re-imaging this host or restoring from a known-good baseline to ensure total remediation of this security incident. 
- Audit affected directories/files for additional suspicious files and remove those found. 
- We recommend taking the following steps to respond to this incident:
 
- Disable SSLVPN until this subset of remediations can be accomplished: 
    - Ensure SonicOS is updated to version 7.3 or greater.
    - Rotate all credentials for all users on the SonicWall SSLVPN. 
    - Audit organization LDAP SSLVPN Default User Groups configuration.
    - Disable/reenable TOTP/MFA for relevant accounts. 
    - You can read more about this set of remediations here: https://www.sonicwall.com/support/notices/gen-7-and-newer-sonicwall-firewalls-sslvpn-recent-threat-activity/250804095336430

All remediations provided can be found in the Huntress Platform: Incident Report: https://tabinc.huntress.io/org/105150/infection_reports/1659489#remediations-tab
