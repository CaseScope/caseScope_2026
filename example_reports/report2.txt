At 2026-01-08 15:20:11 UTC, a Windows Service with the name "ScreenConnect Client (6100c8e237f6b876)" was detected on the endpoint. Evidence indicates prior to this host having the Huntress agent installed, there was a malicious ScreenConnect instance on the host which reaches out to domain: 38[.]69[.]8[.]156 over port 8041. Huntress also observes there were active IP addresses communicating via port 8041, which were 50[.]225[.]154[.]136 and 38[.]69[.]8[.]156.

ScreenConnect is a legitimate remote access tool used by IT professionals for support and management, and because it gives full remote control it can be very useful for authorized administration. However, its same remote access capabilities are frequently leveraged by threat actors—whether via compromised accounts, phishing lures, or trojanized installers—to gain unauthorized control of systems once inside a network. Because of this dual-use nature, its deployment and use should always be authorized and monitored.

We recommend immediately scanning the affected host for additional malware, reviewing network logs for connections to "50[.]225[.]154[.]136 and 38[.]69[.]8[.]156", adding these IPs to your organization's firewall blacklist, and providing security awareness training to the user regarding social engineering tactics. The user's credentials should be considered potentially compromised and may require reset depending on the payload's capabilities. It is recommended to reimage the host from known-good media for full remediation. 

This host has been isolated to prevent further ingress.

Please refer to remediation tasks for next steps.

Threat Descriptions:
--------------------
Rogue ScreenConnect: Huntress has been tracking a number of malicious threat actors convincing users via email into running malicious ScreenConnect (ConnectWise Control) installers that give the threat actor remote access to the host. 

Malicious Remote Management Tool: A legitimate remote management tool has been misused by a threat actor to gain unauthorized access and control of the reported endpoint. Threat actors do this to hide their presence, maintain persistence and further exploit the network.

Persistence : Persistence consists of techniques that adversaries use to keep access to systems across restarts, changed credentials, and other interruptions that could cut off their access. Techniques used for persistence include any access, action, or configuration changes that let them maintain their foothold on systems, such as replacing or hijacking legitimate code or adding startup code.

Prior Compromise: The security incident on this host occurred prior to the Huntress agent installation. We will attempt to address any observed malicious behavior, but there may be unreported changes to this host that will not be detected due to limited telemetry.
