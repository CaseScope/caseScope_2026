Investigative Summary
---------------------
[WARNING]
Please review this incident report to understand what was identified before remediating. There may be unknown malicious processes, files, or other changes made to the host that remain undetected. Restoring from a known good backup or clean OS install is the only way to ensure a complete host-level remediation. 

Huntress detected the following:
- Microsoft SQL Server xp_cmdshell : The MSSQL xp_cmdshell can be utilized by attackers who have valid MSSQL credentials to perform shell/system actions at the privilege level of the MSSQL process, which often has elevated privileges. This can allow attackers to perform unauthorized system modifications, establish a persistent foothold, or deploy malware including ransomware.

Evidence suggests that on '2025-02-03 16:28:06 UTC' enumeration commands were run on host 'ABSRDS' via sqlservr.exe

Logs show that user 'mhastings(S-1-5-21-1402576329-860971221-1604830738-4252)' logged in to the host from source 192.168.1.103(MARYH-5560) at '2025-02-03 16:18:01'. Further evidence shows that xp_cmdshell was enabled at '2025-02-03 16:29:15'

We have isolated this host as a precaution. If this is expected activity, please reject this report. 


Remediation Instructions
------------------------
** Please review these recommendations due to the nature of the attack. **
- Audit any web facing applications that may be interfacing with the MSSQL instance on this server for anomalous activity.
- Rotate the credentials to the MSSQL instance on this server.
- Consider disabling xp_cmdshell in the MSSQL settings if not required for this MSSQL instance.
- Audit user mhastings(S-1-5-21-1402576329-860971221-1604830738-4252) activity.
