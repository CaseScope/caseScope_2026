Hunting overlay selector_key migration runbook
Date: 2026-04-22

Status
Superseded by the single-table event state revert. Hunting, IOC, analyst, and noise reads now use
`events` as the source of truth, so this selector-key migration runbook is retained only as
historical rollout documentation.

Purpose
Safely roll out the materialized events.selector_key column and flip hunting reads to use it.

Success criteria
- Representative live hunting reads for case_id 139 improve materially from the pre-flip live-path
  baseline after the flag flip.
- Post-flip EXPLAIN shows generated SQL referencing e.selector_key directly.
- No correctness regressions in overlay-backed hunting reads.

Abort criteria
- ClickHouse mutation backlog grows without progressing for an extended period during a low-ingest
  window.
- Ingest latency or service health degrades beyond acceptable operational limits.
- Production parity sampling finds selector_key mismatches on live rows.
- Post-flip EXPLAIN still shows the multiIf(...) selector expression instead of e.selector_key.

Reversion rule
- If anything is wrong after the flag flip, first set
  CLICKHOUSE_USE_MATERIALIZED_SELECTOR_KEY=false and restart services.
- Do not leave the flag on if selector_key materialization is incomplete or parity is failing.

[ ] 4. Confirm events table state and selector expression

import importlib.util
import os
import re
import sys
from pathlib import Path
os.environ.setdefault('SECRET_KEY', 'test-secret')
selector_path = Path('/opt/casescope/utils/event_selector.py')
spec = importlib.util.spec_from_file_location('event_selector_under_test', selector_path)
module = importlib.util.module_from_spec(spec)
spec.loader.exec_module(module)
ddl = sys.stdin.read()
match = re.search(r"\bselector_key\b\s+String\s+MATERIALIZED\s+(multiIf\(.*?\))(?=(?:,\n\s*\w+\s+\w+)|\n\)\s*ENGINE|\Z)", ddl, re.DOTALL)
if not match:
    print('NO_SELECTOR_KEY')
    raise SystemExit(0)
live = match.group(1)
canonical = module._raw_selector_expression('')
normalize = lambda value: re.sub(r'\s+', ' ', value).strip()
if normalize(live) == normalize(canonical):
    print('MATCH')
else:
    print('MISMATCH')
    print('--- LIVE ---')
    print(live)
    print('--- CANONICAL ---')
    print(canonical)
    raise SystemExit(1)
PY

[ ] 5. Add the selector_key materialized column
Command:
  clickhouse-client --database casescope --query "
  ALTER TABLE events
  ADD COLUMN IF NOT EXISTS selector_key String MATERIALIZED
  multiIf(
      record_id > 0
          AND source_file IS NOT NULL AND source_file != ''
          AND source_file != '-'
          AND source_host IS NOT NULL AND source_host != ''
          AND source_host != '-',
      concat(
          'record:', toString(record_id),
          '|file:', source_file,
          '|host:', source_host
      ),
      COALESCE(timestamp_utc, timestamp) IS NOT NULL,
      concat(
          'ts:', formatDateTime(COALESCE(timestamp_utc, timestamp), '%Y-%m-%d %H:%i:%S'),
          '|host:', if(ifNull(source_host, '') = '-', '', ifNull(source_host, '')),
          '|artifact:', if(ifNull(artifact_type, '') = '-', '', ifNull(artifact_type, '')),
          '|event:', if(ifNull(event_id, '') = '-', '', ifNull(event_id, '')),
          '|file:', if(ifNull(source_file, '') = '-', '', ifNull(source_file, ''))
      ),
      event_id IS NOT NULL AND event_id != ''
          AND event_id != '-',
      concat('event_id:', event_id),
      ''
  )"
Pass if:
  - ALTER completes successfully
Fail if:
  - ALTER errors
Action on fail:
  - Stop and correct the DDL before proceeding

[ ] 9. Run production parity sampling before enabling the flag
Command:
  python3 - <<'PY'
import importlib.util
import os
import subprocess
from pathlib import Path
os.environ.setdefault('SECRET_KEY', 'test-secret')
repo = Path('/opt/casescope')
selector_path = repo / 'utils' / 'event_selector.py'
spec = importlib.util.spec_from_file_location('event_selector_under_test', selector_path)
module = importlib.util.module_from_spec(spec)
spec.loader.exec_module(module)
rows = subprocess.check_output([
    'clickhouse-client', '--database', 'casescope', '--query',
    '''
    SELECT case_id, event_id, record_id, source_file, source_host,
           formatDateTime(COALESCE(timestamp_utc, timestamp), '%Y-%m-%d %H:%i:%S') AS ts,
           artifact_type, selector_key
    FROM events
    WHERE selector_key != ''
      AND cityHash64(case_id, ifNull(record_id, 0), event_id) % 1000 = 0
    LIMIT 1000
    '''
], text=True).splitlines()
mismatches = 0
for line in rows:
    case_id, event_id, record_id, source_file, source_host, ts, artifact_type, selector_key = line.split('\t')
    expected = module.build_event_selector_key(
        event_id=event_id,
        record_id=record_id,
        source_file=source_file,
        source_host=source_host,
        timestamp=ts,
        artifact_type=artifact_type,
    )
    if expected != selector_key:
        mismatches += 1
        print('MISMATCH', case_id, event_id, record_id, source_file, source_host, ts, artifact_type, selector_key, expected)
        if mismatches >= 20:
            break
print(f'mismatches={mismatches}')
raise SystemExit(1 if mismatches else 0)
PY
Pass if:
  - mismatches=0
Fail if:
  - Any mismatch is reported
Action on fail:
  - Keep feature flag off
  - Investigate data edge cases before proceeding
