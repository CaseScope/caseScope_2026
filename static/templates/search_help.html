🔍 Search Syntax Guide

Master the CaseScope search engine to find exactly what you need during your DFIR investigation

⚡ Quick Reference

term1 term2 AND - both must match anywhere
term1|term2 OR - either can match
"field value" Field:Value pairing
-term Exclude term
(group1)|(group2) OR between groups
4624 Event ID (exact match)

📝 Keyword Searching

The search engine performs case-insensitive substring matching across all indexed fields in each event. This includes usernames, hostnames, file paths, command lines, process names, registry keys, and more.

Basic Examples

powershell

Finds any event containing "powershell" anywhere - in process names, command lines, file paths, etc.

admin

Finds events with "admin" in usernames, group names, or any other field.

192.168.1

Finds all events involving IPs in the 192.168.1.x subnet.

💡 Tip: Searches are case-insensitive. "PowerShell", "powershell", and "POWERSHELL" all return the same results.

🔗 AND Logic (Multiple Terms)

When you enter multiple terms separated by spaces, ALL terms must be present in the event (AND logic). This is useful for narrowing down results.

Examples

powershell encodedcommand

Finds events containing BOTH "powershell" AND "encodedcommand" - often indicates obfuscated PowerShell execution.

4624 network

Finds Event ID 4624 (logon) events that also contain "network" - useful for network logons.

cmd.exe /c net user

Finds events where cmd.exe ran a "net user" command - account enumeration activity.

⚡ OR Logic (Alternatives)

Use the pipe character | (no spaces) to match ANY of the alternatives. This is useful when searching for multiple related terms.

Examples

mimikatz|sekurlsa|lsadump

Finds events containing any Mimikatz-related terms - credential dumping indicators.

psexec|paexec|csexec

Finds events involving any PsExec variants - common lateral movement tools.

4624|4625

Finds both successful (4624) and failed (4625) logon events.

⚠️ Important: No spaces around the pipe! term1|term2 not term1 | term2

🚫 Exclusions

Use the minus sign - to exclude events containing specific terms. Exclusions help filter out noise and focus on suspicious activity.

Examples

4624 -SYSTEM -ANONYMOUS

Finds logon events excluding SYSTEM account and ANONYMOUS LOGON - focuses on real user logons.

powershell -"Windows PowerShell ISE"

Finds PowerShell events but excludes PowerShell ISE - focuses on command-line execution.

4625 -"0xc000006d"

Failed logons excluding "bad username or password" (common noise) - finds more interesting failures.

💡 Tip: Use quotes for multi-word exclusions: -"NT AUTHORITY" to exclude as a phrase.

🎯 Quoted Phrases & Field:Value Pairing

Use quotes to search for exact phrases including spaces. This is especially powerful for field-specific searches because indexed data is stored in FieldName:Value format.

Field-Specific Search Examples

"LogonType 3"

Finds events where LogonType is 3 (Network logon). The quoted phrase matches "LogonType:3" in the indexed data.

"username bob"

Finds events where the username field contains "bob". Matches "username:bob" or "TargetUserName:bob" in indexed data.

4624 "LogonType 10"

Finds Event ID 4624 with LogonType 10 (RDP/RemoteInteractive). The field pairing ensures you're matching the logon type, not just "10" anywhere.

"TargetUserName admin"

Finds events where the TargetUserName field contains "admin" - more precise than just searching "admin" which could match anywhere.

⚠️ Important: Without quotes, 4624 10 would find events containing both "4624" AND "10" anywhere in any field - not specifically logon type 10. Use 4624 "LogonType 10" for precision.

Common Field Names for Pairing

LogonType - Logon type (2=Interactive, 3=Network, 10=RDP)
TargetUserName - Target user in logon events
SubjectUserName - Subject user performing action
IpAddress - Source IP address
WorkstationName - Source workstation
ProcessName - Process executable name
CommandLine - Full command line
TargetFilename - File operation target

🔢 Event ID Search

Pure numeric searches are treated as Event ID lookups, providing exact matches on the event_id field. This is faster and more precise than text searching.

Common Security Event IDs

4624

Successful logon events

4625

Failed logon attempts

4648

Explicit credential logon (RunAs, network share with different creds)

4672

Special privileges assigned - admin logons

4688

Process creation events

4720

User account created

4732

Member added to security group

1102

Audit log cleared - potential anti-forensics

💡 Tip: Combine Event ID with field pairings for precision: 4688 "CommandLine powershell" finds process creation events where the command line contains PowerShell.

📦 Grouping with Parentheses

Use parentheses to create logical groups. Terms within a group are ANDed together, and groups can be ORed using | between them. This is powerful for complex multi-source investigations.

Examples

(4625 security)|(33 vpn failed)

Finds Security log failed logons (4625) OR firewall VPN failures (Event 33) - cross-source auth failure hunting.

(mimikatz)|(lsass.exe dump)|(sekurlsa)

Finds any credential dumping indicators across multiple techniques.

(rdp 4624)|(3389)|(remotedesktop)

Comprehensive RDP session hunting - logon events, port references, or service names.

Per-Group Exclusions

Exclusions inside parentheses apply only to that group. Exclusions outside parentheses apply globally to all results.

(4625 security -anonymous)|(33 vpn -internal)

Security failed logons (excluding anonymous) OR VPN failures (excluding internal IPs). Each exclusion only applies to its group.

(4624 admin)|(4625 admin) -SYSTEM

Admin logon success OR failures, but globally excludes SYSTEM account from all results.

RDP logons (Logon Type 10 = RemoteInteractive). Quoted field pairing ensures we match the logon type field.

Network logons (Type 3) excluding SYSTEM and computer accounts ($ suffix).

Failed logon: user does not exist (user enumeration attack indicator).

📋 Searchable Fields

Keyword searches match against a combined "search blob" that includes all of the following fields:

Source Information

  • source_file, source_path, source_host
  • artifact_type, channel, provider

User Context

  • username, domain, sid
  • logon_type

Process Data

  • process_name, process_path, process_id
  • parent_process, parent_pid
  • command_line

File & Network

  • target_path, file_hash_*
  • src_ip, dst_ip, src_port, dst_port

Registry

  • reg_key, reg_value, reg_data

Detection

  • rule_title, rule_level
  • mitre_tactics, mitre_tags

Start Broad, Then Narrow

Combine search with the time range dropdown to focus on specific windows of interest (incident timeframe).

3️⃣

Cross-Source Hunting

Use groups to search across different log sources: (4625 security)|(vpn failed firewall)

4️⃣

Combine with Filters

Use the event type and alert type filters alongside search for powerful filtering combinations.

Save Your Searches
