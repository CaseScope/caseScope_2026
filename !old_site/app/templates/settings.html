⚙️ Log Level Details

  • DEBUG: Maximum verbosity - Logs all operations, variables, SQL queries, API calls. Use for troubleshooting only (large log files).
  • INFO: Normal operations - Logs important events like case creation, file uploads, user actions. Recommended for production.
  • WARNING: Potential issues - Logs warnings and errors only. Use when disk space is limited.
  • ERROR: Errors only - Logs only critical failures. Use for minimal logging in stable environments.

⚠️ Note: Changes take effect immediately. Restart services if logs are not updating.

🗄️ Case Archiving

Configure cold storage path for archived case files

Absolute path where archived case files will be stored (e.g., spinning disk drive for cold storage)

📦 How Case Archiving Works

  • Archive Process: Creates ZIP of all case files while preserving directory structure
  • Search & Analysis: All indexed data remains searchable (events, IOCs, systems, etc.)
  • IOC Re-Hunting: Fully supported on archived cases (no source files needed)
  • Disabled Operations: Re-index, Re-SIGMA, file uploads (require source files)
  • Restore: Seamlessly extract files back to original locations when needed

💡 Tip: Use a separate drive (e.g., spinning disk) for archive path to maximize SSD space for active cases

⚠️ Requirements

  • Path must be absolute (e.g., /archive not archive)
  • Path must exist and be writable by casescope user and group
  • Sufficient disk space for archived cases (recommend 100GB+ free)
  • Create directory: sudo mkdir -p /archive && sudo chown casescope:casescope /archive
